# Go Notes API

REST API сервис для управления заметками, написанный на Go.

## О проекте

Этот проект представляет собой реализацию простого RESTful API на Go. Его основная цель — демонстрация навыков бэкенд-разработки, включая работу с HTTP, реляционными базами данных, контейнеризацией и применение конкурентности для решения практических задач.

## Стек технологий

*   **Язык:** Go
*   **HTTP-роутер:** `gorilla/mux`
*   **Веб-сервер:** Стандартная библиотека `net/http`
*   **База данных:** PostgreSQL
*   **Драйвер для БД:** `jackc/pgx`
*   **Контейнеризация:** Docker, Docker Compose

## Структура и функционал

### API эндпоинты

Сервис реализует стандартные CRUD-операции для сущности "заметка".

| Метод    | Путь           | Описание                  |
| -------- | -------------- | ------------------------- |
| `POST`   | `/notes`       | Создание новой заметки.   |
| `GET`    | `/notes`       | Получение списка заметок. |
| `GET`    | `/notes/{id}`  | Получение заметки по ID.  |
| `PUT`    | `/notes/{id}`  | Обновление заметки по ID. |  
| `DELETE` | `/notes/{id}`  | Удаление заметки по ID.   |

### Хранение данных

Данные хранятся в PostgreSQL, развернутым в docker.

### Логирование

Для каждого входящего HTTP-запроса реализовано логирование через `middleware`. Запись логов вынесена в отдельную горутину для предотвращения блокировки основного потока обработки запроса.

## План реализации

1.  **Настройка окружения:** Инициализация проекта, создание `docker-compose.yml` для PostgreSQL.
2.  **База приложения:** Настройка роутера `gorilla/mux` и подключения к базе данных.
3.  **Логирование:** Реализация `middleware` для логирования запросов и асинхронной записи в файл.
4.  **Реализация API:** Написание обработчиков для всех CRUD-эндпоинтов.
